Fix Row Level Security issues for gallery uploads

RLS Policy Issue

The "new row violates row-level security policy" error occurs because Supabase has Row Level Security enabled on the gallery_images table.

To fix this, you need to:

  1. Go to your Supabase Dashboard
  2. Navigate to Authentication > Policies
  3. Find the gallery_images table
  4. Add a policy that allows INSERT operations
  5. Or temporarily disable RLS for admin operations

Quick SQL Fix (run in Supabase SQL Editor):

-- Option 1: Allow all operations (less secure)
ALTER TABLE gallery_images DISABLE ROW LEVEL SECURITY;

-- Option 2: Add specific policy for inserts
CREATE POLICY "Allow gallery image inserts" ON gallery_images
FOR INSERT WITH CHECK (true);

-- Option 3: Allow all operations for authenticated users
CREATE POLICY "Allow all gallery operations" ON gallery_images
FOR ALL USING (auth.role() = 'authenticated');
