Dashboard Setup Instructions

⚠️ Important: The admin dashboard must be served through a web server, not opened directly as a file, to avoid CORS issues with Supabase.

How to Run the Dashboard Locally

Option 1: Using Python (Recommended)

If you have Python installed:

# Navigate to the dashboard folder
cd dashboard

# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000

Then open: http://localhost:8000/admin-dashboard.html

Option 2: Using Node.js

If you have Node.js installed:

# Install a simple HTTP server
npm install -g http-server

# Navigate to the dashboard folder
cd dashboard

# Start the server
http-server -p 8000

Then open: http://localhost:8000/admin-dashboard.html

Option 3: Using Live Server (VS Code Extension)

If you're using VS Code:

  1. Install the "Live Server" extension
  2. Right-click on admin-dashboard.html
  3. Select "Open with Live Server"

Option 4: Using PHP

If you have PHP installed:

# Navigate to the dashboard folder
cd dashboard

# Start PHP server
php -S localhost:8000

Then open: http://localhost:8000/admin-dashboard.html

✅ Why this is necessary:
Modern browsers block requests from local files (file://) to external APIs like Supabase for security reasons. Serving the files through a local web server (http://localhost) resolves this CORS issue.

Testing the Connection

Once you have the dashboard running through a web server, you can also test the Supabase connection using:

http://localhost:8000/test-connection.html

Troubleshooting

  • Port 8000 already in use: Try a different port like 8080 or 3000
  • Python not found: Make sure Python is installed and in your PATH
  • Still getting CORS errors: Make sure you're accessing via http://localhost, not file://
  • Supabase connection fails: Check your internet connection and Supabase project status
